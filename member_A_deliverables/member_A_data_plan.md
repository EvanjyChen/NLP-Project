# Member A Plan: Data Preparation and Exploratory Analysis

## Objective

Provide a verified, leakage-safe TSAR 2025 dataset interface for model development and
evaluation, then produce the exploratory data analysis required by the preliminary
paper.

Member A's first priority is not completing every visualization. It is delivering a
correct dataset schema and a few validated examples so that the model and evaluation
owners can begin work.

## Deliverables

1. A local copy of the official TSAR 2025 data and evaluation resources.
2. `data_schema.md`, documenting every field and its meaning.
3. `split_policy.md`, stating which examples may be used for prompts or development.
4. A validation script that checks counts, required fields, CEFR labels, duplicates,
   empty text, and malformed records.
5. A normalized machine-readable dataset for the model and evaluation pipelines.
6. An EDA script that regenerates all reported statistics and figures.
7. A paper-ready dataset statistics table and one useful EDA figure.
8. A short, fact-checked Data section draft.
9. A handoff note describing paths, commands, known limitations, and unresolved issues.

## Phase 2: Define the Shared Data Contract

**Target: first or second working day**

Normalize each generation request into a stable format such as:

```json
{
  "id": "example_001_A2",
  "source_id": "example_001",
  "source": "Original paragraph",
  "source_level": "C1",
  "target_level": "A2",
  "references": ["Reference simplification"],
  "split": "trial"
}
```

Define the prediction format with Sankeerth Adisha (B) and David Kim (C):

```json
{
  "id": "example_001_A2",
  "system": "llama_zero_shot",
  "output": "Generated simplification",
  "seed": 42,
  "revision_count": 0
}
```

### Validation Checks

- Unique request IDs
- Required fields present
- No empty source/reference text
- CEFR labels restricted to expected values
- Correct source-to-target pairing
- No duplicate source-target requests
- Valid UTF-8 and normalized line endings
- Trial/test overlap check
- Prompt-example and evaluation-example overlap check

### Exit Criteria

- Sankeerth Adisha (B) and David Kim (C) successfully consume the normalized sample.
- The validator fails clearly when a required field is removed or corrupted.
- The split policy prevents using evaluation examples as few-shot demonstrations.

## Phase 3: Exploratory Data Analysis

**Target: days 2-3**

Calculate statistics separately for source texts, A2 references, and B1 references:

- Number of texts and generation requests
- Source and target CEFR distribution
- Word-count mean, median, standard deviation, minimum, and maximum
- Sentence-count mean and median
- Average word length
- Type-token ratio or another clearly defined lexical-diversity measure
- FKGL distribution, labeled as descriptive rather than a CEFR classifier
- Source-to-reference compression ratio
- Number of sentence splits or merges where measurable

Compare A2 and B1 references using paired differences where the data permits:

- Reference length
- Sentence count
- FKGL
- Compression ratio

### Paper Outputs

Create one compact table:

| Group | N | Words | Sentences | FKGL | Compression ratio |
|---|---:|---:|---:|---:|---:|
| Sources | | | | | |
| A2 references | | | | | |
| B1 references | | | | | |

Create one figure:

- Preferred: paired source/reference word-count distributions separated by A2 and B1.
- Alternative: compression-ratio distributions by target CEFR.

Avoid decorative figures and do not include plots that repeat the table without
revealing a pattern.

### Exit Criteria

- Every number in the table is generated by the EDA script.
- Figure labels, units, sample sizes, and captions are complete.
- Outliers are inspected rather than silently removed.
- The analysis identifies at least two observations relevant to model design.

## Risks and Responses

| Risk | Response |
|---|---|
| Official data cannot be downloaded | Contact instructors/task organizers immediately and document the request |
| Release schema differs from the task paper | Treat the downloaded release as operational truth and document the discrepancy |
| Test references are unavailable | Use permitted trial data for preliminary development and avoid inventing results |
| Too few examples for strong statistical claims | Report uncertainty and paired instance-level differences |
| Few-shot leakage | Enforce the split policy programmatically |
| FKGL conflicts with CEFR predictions | Report both as different constructs; do not use FKGL as a CEFR substitute |
| EDA output changes between runs | Pin package versions and make preprocessing deterministic |
